[id="security-and-compliance_{context}"]
== Security and Compliance

All {rh-rhacscs-first} data in the Central instance is encrypted in transit and at rest.
The data is stored in secure storage with full replication and high availability together with regularly scheduled backups.
The {product-title-managed-short} is available through cloud data centers that ensure optimal performance and the ability to meet data residency requirements.

[id="authentication-provider_{context}"]
=== Authentication provider

When you create a Central instance using link:https://console.redhat.com/[Red Hat Hybrid Cloud Console], authentication for the cluster administrator is configured as part of the process.
Customers must manage all access to the Central instance as part of their integrated solution.
For more information about the available authentication methods, see xref:../operating/manage-user-access/understanding-authentication-providers.adoc#understanding-authentication-providers[Understanding authentication providers].

The default identity provider in {product-title-managed-short} is Red Hat Single Sign-On (SSO). For more information about authentication using Red Hat SSO, see xref:../cloud_service/getting-started-rhacs-cloud-ocp.adoc#default-access-acs-console[Default access to ACS Console].

=== Regulatory compliance
